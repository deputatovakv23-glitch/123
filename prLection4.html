Архитектура информационной системы "Интернет-магазин"

УП.05: Разработка модели архитектуры ИС

Цель проекта

Спроектировать и отобразить графически основные компоненты системы, их взаимодействие и место в общей структуре.

Тип архитектуры

Для веб-приложений, таких как интернет-магазин, используется трехуровневая архитектура, обеспечивающая модульность, масштабируемость и безопасность.

Уровень 1: Presentation Layer

Клиентская часть (интерфейс в браузере)

Уровень 2: Business Logic Layer

Серверная часть (основная логика системы)

Уровень 3: Data Access Layer

Уровень данных (хранение информации)

Модели архитектуры

1Контекстная диаграмма

Показывает взаимодействие с внешними системами и субъектами.

  • Покупатель
  • Платежный шлюз
  • Курьерская служба
  • Администратор

2Компонентная диаграмма

Отображает основные программные модули и связи между ними.

  • Frontend
  • Backend (API Gateway, модули)
  • Уровень данных

3Физическая архитектура

Показывает развертывание компонентов на серверах.

  • Балансировщик нагрузки
  • Веб-серверы
  • Серверы приложений
  • Кластер БД

4Модель данных

Отображает структуру базы данных.

  • Сущности и их атрибуты
  • Связи между сущностями
  • ER-диаграмма

Ключевые компоненты системы

Frontend (Клиентское приложение)

  • Интерфейс пользователя (React, Vue.js)
  • Адаптивный дизайн
  • Клиентская валидация

Backend (Серверное приложение)

  • API Gateway - единая точка входа
  • Модуль аутентификации (Auth)
  • Модуль каталога (Catalog)
  • Модуль заказов (Order)
  • Модуль оплаты (Payment)
  • Модуль уведомлений (Notification)

Уровень данных

  • База данных (PostgreSQL/MySQL)
  • Кеш (Redis)
  • Файловое хранилище (S3)

Оформление отчета по УП.05

  • Введение: Обоснование выбранного типа архитектуры
  • Диаграммы: Все 4 модели с пояснениями компонентов
  • Связь с требованиями: Как архитектура удовлетворяет функциональным и нефункциональным требованиям
  • Вывод: Резюме о оптимальности предложенной архитектуры

Эта работа демонстрирует понимание не только того, что делать, но и как это должно быть реализовано на практике.
